PHAN 4: MO PORT MANG (NETWORK SECURITY GROUP)
------------------------------------------------
Luc tao VM chi mo port 22. Sau khi tao xong, nguoi dung tu vao:
  VM "vsl-project" -> Networking -> Inbound port rules -> Add inbound
  port rule, them 2 rule:

  Rule 1: Destination port 8080, Protocol TCP, Source Any,
          Priority 310, Name "Allow-Backend-8080"
          -> cho phep truy cap Spring Boot API tu internet

  Rule 2: Destination port 9000, Protocol TCP, Source Any,
          Priority 320, Name "Allow-MinIO-9000"
          -> BAT BUOC phai mo cong khai vi trinh duyet load video/anh
          TRUC TIEP tu MinIO (khong qua backend proxy), thong qua bien
          MINIO_PUBLIC_ENDPOINT

  KHONG mo port 9001 (MinIO console quan tri) va 5555 (pgAdmin) ra
  internet - day la trang quan tri, de lo ra ngoai rat rui ro.

PHAN 10: THIET LAP AUTO-DEPLOY TU GITHUB ACTIONS (CD)
------------------------------------------------------------
10.1. Tao 1 CAP SSH KEY MOI RIENG cho CI (KHONG dung lai key admin
      .pem ca nhan), de neu secret nay lo thi chi can xoa 1 dong trong
      authorized_keys tren VM ma khong anh huong key admin chinh:
        ssh-keygen -t ed25519 -f vsl-ci-deploy-key -N ""
        -> tao ra vsl-ci-deploy-key (private) va .pub (public)

10.2. Them public key vao VM:
        cat vsl-ci-deploy-key.pub >> VM:~/.ssh/authorized_keys
      Test lai bang key moi de chac chan hoat dong truoc khi dung
      trong CI.

10.3. Sua file .github/workflows/backend-ci-cd.yml: them job "deploy"
      moi, chay SAU job "build-and-push" (needs: build-and-push), dung
      action "appleboy/ssh-action@v1.2.0":

        deploy:
          name: Deploy to Azure VM
          needs: build-and-push
          runs-on: ubuntu-latest
          steps:
            - name: Deploy via SSH
              uses: appleboy/ssh-action@v1.2.0
              with:
                host: ${{ secrets.VM_HOST }}
                username: ${{ secrets.VM_USER }}
                key: ${{ secrets.VM_SSH_KEY }}
                script: |
                  cd ~/vsl-backend
                  docker compose pull app
                  docker compose up -d app
                  docker image prune -f

      Chi pull/restart service "app" (--/khong dung "--no-deps" nhung
      vi cau hinh postgres/minio khong doi nen compose khong dong lai
      2 container do), tranh downtime khong can thiet cho DB/MinIO.

10.4. Them 3 GitHub Actions Secrets (Settings -> Secrets and variables
      -> Actions -> New repository secret) - nguoi dung tu them vi day
      cung la quyen admin repo:
        VM_HOST    = 20.48.61.15
        VM_USER    = azureuser
        VM_SSH_KEY = (toan bo noi dung file vsl-ci-deploy-key, ca dong
                     BEGIN/END)

10.5. Commit + push:
      - Dang o nhanh "dev": git add, git commit, git push origin dev
      - Job "deploy" trong workflow chi kich hoat khi push vao "main"
        (dieu kien if: github.ref == 'refs/heads/main'), nen phai dua
        thay doi len main:
          git checkout main
          git merge dev --ff-only   (fast-forward, khong tao merge
                                      commit vi main dang la to tien
                                      truc tiep cua dev)
          git push origin main

10.6. Xac nhan deploy tu dong chay dung: theo doi container "vsl-
      backend" tren VM qua SSH, thay image SHA doi tu
      sha256:26df38ea... sang sha256:68895605... va container duoc
      tao lai (STARTED thoi diem moi), roi chuyen "healthy", va van
      truy cap duoc tu ben ngoai qua curl.
